For a while now I’ve not been able to connect to the Purdue campus LDAP server from Thunderbird. Today I got a bit more determined to figure out the issue, and discovered after about 5 hours that Thunderbird is just fucked in the following situation:

1) You are running WinXP 2) You have the Windows IPv6 stack installed 3) You try to connect to an LDAP server from the Thunderbird Address Book

This guy does a better job of explaining it than I ever could, and I only half understood what the fuck was going on anyway, but nevertheless, this is beyond gay. All the other apps with LDAP server query functionality (like Ye Olde Windows Address Book) work fine. I actually ended up installing Windump on my machine, and saw that while every other app connected to the LDAP server fine, Thunderbird’s packets went off on a fucking trip through Singapore and got sold into sex slavery.

Anyway, the workaround — until the Thunderbird guys fix this 2+ year old bug — is to go into the properties for your ethernet device and uninstall Microsoft Version 6 TCP/IP (or whatever they call it). I guess I wasn’t using it anyway…
